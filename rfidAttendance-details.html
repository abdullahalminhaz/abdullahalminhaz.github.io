RFID & Google Sheets — Smart Attendance

NodeMCU (ESP8266) + RC522 · Real-time logging to Google Sheets

Aug 2025 • Engineering Project Design Course — JSTU

Project Overview

A compact, portable attendance system that identifies cardholders using an RC522 RFID reader and logs name, date, and time instantly to a Google Sheet via Wi-Fi. The NodeMCU (ESP8266) hosts the networking code, fetches time via NTP for accurate timestamps, and pushes attendance records to a Google Apps Script endpoint. The local LCD provides instant feedback to users while the cloud sheet stores records for reporting and analysis.

Practical Demonstration

The demo shows card detection flow, LCD feedback, successful HTTP logging, and timestamp verification. The system handles URL encoding of names, retries on failed network attempts, and displays clear success/error messages. Use the video to inspect read range, response latency, and how the NodeMCU handles intermittent Wi-Fi.

Practical notes: The system is designed to be lightweight and extendable — later additions can include alerting, RFID database management, or face-ID integration.

Key Features

RFID Identification

Reads UID and maps card IDs to user names for instant recognition.

Cloud Logging

HTTP GET to Google Apps Script writes name, date, and accurate NTP timestamp to Sheets.

Local Feedback

I2C LCD displays the cardholder name and success/failure information instantly.

Hardware & Software

Hardware
  • NodeMCU (ESP8266) — network & controller
  • RFID RC522 module
  • LiquidCrystal I2C LCD (16×2)
  • 5V power supply / battery (portable option)
  • Breadboard / PCB and wiring harness
Software & Libraries
  • Arduino IDE (C/C++)
  • Google Apps Script for Sheets endpoint
  • NTPClient library for accurate time
  • WiFiClient / HTTP GET logic + URL encoding
  • Small retry logic & watchdog to manage memory

Challenges & Learning

The main obstacles were URL encoding of names, transient network failures, and tight memory constraints on the ESP8266. Solving these taught practical lessons about HTTP handling, Google Apps Script backend flows, time synchronization, and memory-aware programming on microcontrollers.

Applications

  • Classroom and lab attendance automation
  • Office / lab entry logging and audit
  • Library check-in systems
  • Events & workshops attendance tracking
  • Access control with simple logging (can be expanded)
